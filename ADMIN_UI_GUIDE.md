## 🔗 API 端点

### 管理员端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/admin/google-ads-mcc` | GET | 获取 MCC 列表 |
| `/api/admin/google-ads-mcc` | POST | 创建 MCC 配置 |
| `/api/admin/google-ads-mcc/[id]` | DELETE | 删除 MCC 配置 |
| `/api/admin/google-ads-mcc/[id]/authorize` | GET | 生成 MCC OAuth URL |
| `/api/admin/google-ads-mcc/callback` | GET | MCC OAuth 回调 |
| `/api/admin/google-ads-mcc/[id]/bindings` | GET | 获取 MCC 绑定用户列表 |
| `/api/admin/users` | GET | 获取用户列表 |
| `/api/admin/user-mcc-binding` | GET | 检查用户授权状态 |
| `/api/admin/user-mcc-binding` | POST | 绑定用户到 MCC |
| `/api/admin/user-mcc-binding/[userId]` | DELETE | 解除用户绑定 |
| `/api/admin/user-mcc-bindings` | GET | 获取所有用户绑定 |

### 用户端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/google-ads/oauth` | GET | 生成用户 OAuth URL |
| `/api/google-ads/oauth/callback` | GET | 用户 OAuth 回调 |

## 🚀 使用流程

### 管理员配置 MCC

1. 访问 `/admin/google-ads-mcc`
2. 点击"新增 MCC 配置"
3. 填写 MCC 信息：
   - MCC Customer ID (10 位数字)
   - OAuth Client ID
   - OAuth Client Secret
   - Developer Token
4. 点击"保存配置"
5. 点击"启动 OAuth 授权"
6. 在新窗口完成 Google 授权
7. 返回页面，状态更新为"已授权"

### 管理员绑定用户

**方式 1: 从 MCC 管理页面**
1. 在 MCC 列表中选择一个 MCC
2. 点击"+ 绑定新用户"
3. 跳转到用户绑定页面

**方式 2: 从用户管理页面**
1. 访问 `/admin/users`
2. 从左侧选择用户
3. 在右侧填写绑定信息：
   - 选择 MCC 账号
   - 填写 Customer ID
4. 点击"绑定用户"

### 查看绑定状态

**MCC 维度**:
1. 访问 `/admin/google-ads-mcc`
2. 选择一个 MCC
3. 查看下方的"绑定用户列表"

**用户维度**:
1. 访问 `/admin/users`
2. 选择一个用户
3. 查看"绑定详情"

### 解除绑定

1. 访问 `/admin/users`
2. 选择已绑定的用户
3. 在"绑定详情"卡片点击"解除绑定"
4. 确认操作

### 删除 MCC

⚠️ **警告**: 删除 MCC 将解除所有用户绑定！

1. 访问 `/admin/google-ads-mcc`
2. 找到要删除的 MCC
3. 点击右侧的删除图标 🗑️
4. 确认删除

## 📊 状态说明

### MCC 授权状态

| 状态 | 标识 | 说明 |
|------|------|------|
| 已授权 | ✓ 绿色 | MCC 已完成 OAuth 授权，可绑定用户 |
| 未授权 | ⚠ 黄色 | MCC 未完成 OAuth 授权，需先授权 |

### 用户授权状态

| 状态 | 标识 | 说明 |
|------|------|------|
| 已授权 | ✓ 绿色 | 用户已完成 OAuth 授权，可使用广告功能 |
| 需要重新授权 | ⚠ 黄色 | 配置变更或 token 过期，需重新授权 |
| 未授权 | ○ 灰色 | 用户未完成 OAuth 授权 |

## 🔐 权限控制

- **管理员** (`role='admin'`): 可访问所有管理页面和 API
- **普通用户**: 只能访问用户授权页面 `/google-ads/authorize`

## 📝 常见问题

### Q: 为什么看不到用户列表？

A: 检查以下几点：
1. 确认已登录且有管理员权限
2. 检查 `/api/admin/users` API 是否正常
3. 查看浏览器控制台错误

### Q: 绑定用户时提示"MCC 未授权"？

A: 需要先在 MCC 管理页面完成 MCC OAuth 授权。

### Q: 如何知道用户是否需要重新授权？

A: 在用户绑定详情中查看状态：
- "需要重新授权"（黄色）→ 用户需重新完成 OAuth
- "已授权"（绿色）→ 用户可正常使用

### Q: 删除 MCC 后用户会怎样？

A: 用户的绑定关系会被标记为"需要重新授权"，token 会被清除。用户需要重新绑定到新的 MCC 并完成授权。
